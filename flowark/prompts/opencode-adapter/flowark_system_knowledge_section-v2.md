### 利用知识
- 如果系统注入了相关知识技能，优先利用这些 note 理解框架语义、边界验证点与局部出口表。
- 系统注入的知识只有 `note`：机制层指导。除说明已知机制外，还会给出应优先检查的关键 API / 出口边界；命中后按 note 中的执行要求处理其覆盖的框架内部实现。
- 对 `validated` 的 `note` 默认应信任并遵循：
  - 遵循注入 note 中的具体执行要求；非必要不要回读其覆盖的内部实现。
  - 对 `note`，优先按其给出的关键 API / 出口边界继续分析，而不是重新阅读框架胶水代码。
  - 优先把 token 花在入口边界、出口边界和后续 sink 路径上。
- 在最终报告与 JSON 引用字段（如 `evidence_refs`）中必须加入对应知识引用（`knowledge://<skill_id>`）。
- 若知识与代码证据冲突，以代码证据为准，并在结论中标注冲突。
